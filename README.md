# Сеть организаций РФ — Network Analysis Pipeline

Инструмент для построения интерактивных карт связей между организациями России на основе данных ЕГРЮЛ.

---

## Быстрый старт (без API-ключа, синтетические данные)

```bash
# 1. Клонировать / распаковать проект
cd org_network

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Запустить демо-анализ Газпрома (терминал)
python example_analysis.py

# 4. Запустить веб-интерфейс
streamlit run app.py
```

Открыть в браузере: http://localhost:8501 → нажать «Загрузить демо-данные».

---

## Получение API-ключа DaData (реальные данные ЕГРЮЛ)

1. Перейти на https://dadata.ru/
2. Нажать «Войти» → «Зарегистрироваться» (бесплатно)
3. После входа: «Настройки» → «API-ключи» → скопировать **Секретный ключ**
4. Лимит: **10 000 запросов/сутки** бесплатно (достаточно для ~3 000 организаций/день с учётом rate limit)

Установить ключ (один из способов):

```bash
# Способ 1: переменная окружения
export DADATA_TOKEN="ваш_ключ_здесь"
python example_analysis.py

# Способ 2: в коде fetcher.py (строка 14)
DADATA_TOKEN = "ваш_ключ_здесь"

# Способ 3: через веб-интерфейс
streamlit run app.py
# → поле «DADATA_TOKEN» в боковой панели
```

## Пример запуска для ПАО «Газпром» (ИНН 7736050003)

```python
from graph_builder import build_ego_graph
from metrics import enrich_graph_with_metrics, top_nodes_by_pagerank
from visualizer import export_to_html_d3

# Строим ego-граф глубиной 2
G = build_ego_graph("7736050003", depth=2)
G = enrich_graph_with_metrics(G)

# Топ организаций по PageRank
for rank, (inn, score, label) in enumerate(top_nodes_by_pagerank(G, 5), 1):
    print(f"#{rank} {label}: {score:.4f}")

# Экспорт
export_to_html_d3(G, "gazprom.html")
```

### Типы рёбер графа

| Тип                | Цвет   | Значение                                    |
|--------------------|--------|---------------------------------------------|
| `parent_child`     | 🔴 красный | Одна организация — учредитель другой    |
| `common_founder`   | 🟠 оранжевый | Общий учредитель (юрлицо или физлицо) |
| `common_director`  | 🟢 зелёный | Общий руководитель (по ФИО)             |
| `same_industry`    | 🔵 синий | Совпадение ОКВЭД (2-значный префикс)      |

## Известные ограничения

### 1. DaData не возвращает всех учредителей
DaData показывает только данные ЕГРЮЛ — **юридически значимые записи**. Реальная цепочка владения через офшоры, номинальных держателей и трасты недоступна через открытые API. Для глубокого анализа бенефициарного владения нужны:
- Прямые выгрузки ФНС (закрытый доступ, коммерческие операторы)
- Базы типа Spark-Interfax, Kontур.Фокус (платные, ~500 $/мес)

### 2. Физические лица без ИНН
DaData не всегда возвращает ИНН физических лиц-учредителей. Мы идентифицируем их по ФИО (`common_director`, `common_founder`), но коллизии возможны.

### 3. Масштаб
- Proof of Concept: 50–300 организаций (разумная скорость без кэша)
- Производственный вариант: нужна БД (SQLite/PostgreSQL) и очередь задач (Celery)
- Полный ЕГРЮЛ: ~6 млн ЮЛ → требует прямого доступа к XML-выгрузкам ФНС (~100 ГБ)

### 4. Актуальность данных
DaData синхронизируется с ФНС раз в сутки. Свежие изменения в составе учредителей могут не отображаться.
